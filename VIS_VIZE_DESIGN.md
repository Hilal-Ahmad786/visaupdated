# VİS VİZE — Website & Admin Panel Design System

## 1. Document Purpose

This file is the single source of truth for designing the complete **VİS VİZE RANDEVU HİZMETLERİ** website and its admin panel in Stitch.

The redesign must:

- Replace the current visual identity and old website appearance.
- Use the uploaded VİS VİZE logo as the primary brand reference.
- Prioritize conversions from Google Ads traffic.
- Keep the phone number and lead form highly visible.
- Make the main hero area form-focused.
- Give special importance to the text above and around forms.
- Keep a direct click-to-call action visible near the bottom of important landing pages.
- Build a premium, trustworthy, modern, and mobile-first experience.
- Include a full admin panel so website content can be managed without changing code.

The project should use the conversion logic of successful visa lead-generation websites, but it must **not copy their visual design**.

# 3. Core Business Goal

The primary purpose of the website is not only to provide information. It is to generate qualified visa-service leads.

The website must guide visitors toward one of two primary conversion actions:

1. **Call now**
2. **Complete the pre-application form**

Secondary conversion actions:

- Start WhatsApp conversation
- Request an appointment
- Select a destination country
- View required documents
- Read a visa guide
- Return to an unfinished form

Every major page should clearly answer:

- What service is offered?
- Who is it for?
- Why should the visitor trust the company?
- What information is needed to begin?
- How can the visitor call immediately?
- What happens after submitting the form?

# 4. Conversion Strategy

## 4.1 Conversion Priority

Use this action hierarchy throughout the website:

### Primary CTA

**Ücretsiz Ön Başvuruyu Başlat**

### Secondary CTA

**Hemen Ara**

### Tertiary CTA

**WhatsApp’tan Yaz**

The call action and form action should receive the strongest visual emphasis.

WhatsApp should remain available, but it should not visually compete with the call and form actions unless the client later decides otherwise.

## 4.2 Above-the-Fold Rule

On homepage and paid-ad landing pages, the first screen must include:

- Brand identity
- Clear outcome-focused headline
- Short supporting explanation
- Trust indicator
- Visible phone number
- Click-to-call button
- Short form or the first step of a multi-step form

The visitor should not need to scroll before seeing how to call or begin the form.

## 4.3 Form-First Hero

Desktop hero layout:

- Left side: headline, supporting text, trust points, phone number
- Right side: elevated pre-application form card
- Optional background visual: subtle international travel or passport-related image
- Avoid a large decorative image that pushes the form below the fold

Mobile hero layout:

1. Short headline
2. One-sentence supporting text
3. Click-to-call button
4. Form card
5. Trust badges

## 4.4 Sticky Mobile Conversion Bar

On mobile, show a fixed bottom action bar on all important public pages:

- Left button: **Hemen Ara**
- Right button: **Ön Başvuru**

The sticky bar should:

- Remain visible without covering important content
- Respect device safe areas
- Use icons and clear labels
- Hide on the thank-you page
- Be configurable from admin settings

## 4.5 Landing Page Call Section

Near the bottom of country and service landing pages, include a high-contrast section:

**Vize sürecinizle ilgili hemen bilgi alın**

Supporting text:

**Uzman danışmanlarımız başvuru türünüzü değerlendirerek size uygun süreci açıklasın.**

Actions:

- Large visible phone number
- **Tıkla ve Ara**
- **Ön Başvuru Formunu Doldur**

# 14. Form Design System

## 14.1 Main Pre-Application Form

Use a multi-step form to reduce perceived effort.

### Step 1 — Travel Need

- Destination country
- Application purpose

### Step 2 — Applicant

- First name
- Last name
- Phone number
- Email

### Step 3 — Details

- Residence city
- Number of applicants
- Planned travel date
- Optional message

### Step 4 — Consent and Submit

- Summary
- KVKK consent
- Commercial communication consent where required
- Submit button

The exact steps must be editable from the admin panel.

## 14.2 Form Header Copy

The text above the form is a conversion-critical area.

Recommended structure:

**Ücretsiz Vize Ön Değerlendirmenizi Başlatın**

**Kısa formu doldurun. Uzman ekibimiz başvuru türünüzü değerlendirerek size en kısa sürede ulaşsın.**

Small reassurance row:

- 2–3 dakikada tamamlanır
- Güvenli bilgi aktarımı
- Ücretsiz ilk değerlendirme

## 14.3 Form Card

The form card should include:

- Strong title
- Short explanatory copy
- Progress indicator
- Clear labels
- Large controls
- Inline validation
- Back and next buttons
- Phone alternative
- Privacy reassurance

## 14.4 Form Behaviors

- Save entered fields while moving between steps
- Preserve form data during accidental refresh where practical
- Country can be preselected from landing-page URL
- Show clear error messages
- Prevent duplicate submissions
- Display submitting state
- Track form start, step completion, and successful submission
- Redirect to thank-you page after confirmed submission

## 14.5 Form Security UX

Use silent protections that do not disturb real visitors:

- Honeypot field
- Minimum completion time
- Server-side validation
- Rate limiting
- Duplicate submission detection

Do not show CAPTCHA to every visitor by default.

# 15. Trust Components

Use verifiable trust signals only.

Possible trust elements:

- Years of experience
- Number of supported countries
- Service areas
- Real office address
- Real customer reviews
- Transparent process
- Secure form
- Response time
- Relevant business registration information
- Clear legal disclaimer

Avoid guaranteed approval language.

Use wording such as:

- “Profesyonel başvuru desteği”
- “Evrak hazırlama ve süreç takibi”
- “Başvurunuza uygun yol haritası”
- “Uzman ekip desteği”

Avoid:

- “Kesin vize”
- “%100 onay”
- “Garantili randevu”
- Unsupported approval percentages

- Phone number remains visible in the header.
- Form card width: approximately 420–480px.

## Tablet

- Form may stack below copy.
- Keep phone CTA visible.
- Country cards become two columns.

## Mobile

- Shorter hero text.
- Form card occupies full width.
- Sticky call/form bar.
- One-column cards.
- Filters become horizontal scroll or bottom sheet.
- Tables transform into stacked records.
- Do not place long paragraphs before the form.

---

# 26. Motion

Use motion only to improve clarity.

Allowed:

- Button hover
- Form step transition
- Accordion open/close
- Card hover
- Header shadow on scroll
- Success check animation
- Subtle number count-up

Avoid:

- Parallax
- Constant floating objects
- Flashing buttons
- Aggressive popups
- Auto-rotating carousels

---

# 27. Copywriting Direction

Customer-facing content should be in Turkish.

Tone:

- Clear
- Reassuring
- Direct
- Respectful
- Professional
- Easy to scan

Use action-oriented headings.

Examples:

- “Başvurunuza Uygun Süreci Birlikte Planlayalım”
- “Hangi Ülkeye Başvurmak İstiyorsunuz?”
- “Evraklarınızı Başvuru Türünüze Göre Hazırlayın”
- “Uzman Danışmanlarımız Sizi Arasın”
- “Vize Süreciniz Hakkında Hemen Bilgi Alın”

Avoid fear-based copy and guaranteed-result language.

---

# 28. Stitch Output Requirements

For every screen designed in Stitch:

- Use this design system consistently.
- Use the uploaded logo.
- Use real Turkish interface copy.
- Create high-fidelity UI, not a wireframe.
- Show desktop at 1440px width.
- Also show mobile at 390px width for conversion-critical pages.
- Include complete page sections, not only the hero.
- Use realistic form states.
- Use reusable components.
- Keep the form and phone prominent.
- Do not copy the visual design of reference websites.
- Follow their conversion logic only.
- Keep public website and admin panel visually related but functionally distinct.
- Ensure every screen can later be implemented in Next.js.
